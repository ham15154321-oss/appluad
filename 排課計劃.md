# 赫綵「衝班計劃・課程預排」心法

來源：2026/5/21 桃區課程預排會議 P1、課程預排技巧教育訓練、招生衝班流程圖（客服主導版）。
適用對象：學院主管、產品部、客服、業務。

## 一、核心理念：排課權限下放

- 排課與老師遴選權力，由產品部主導 **交還給各單位主管**。
- 主管必須具備「**課程邏輯**」思維，配合招生策略做正確判斷。
- 主管應有 **3-4 個月預判能力**：5 月就要規劃 9 月、10 月的課程，預留足夠招生週期。

## 二、ABC 課程體系與漏斗效應

- 課程分級：A（基礎）→ B / C / D（進階 / 系列）。
- 漏斗比例：**4 個 A → 2 個 B → 1 個 C → 1 個 D**。
- 開班門檻：至少 **3 個 A 班（或 2 個，每班約 20 人）才能支撐 1 個 B 班**，確保人數基數。
- 沒有足夠 A 類學生，B 班就開不起來 — 先養 A，再衝 B。

## 三、課程預排七大要素

| 要素 | 內容 |
|---|---|
| 課程類型 | 商用視覺、數位繪畫、攝影課程、影視特效、遊戲設計、音樂創作、室內設計、單科 |
| 課程順序 | A、B、C（依漏斗邏輯排序） |
| 教室選擇 | PC、MAC、錄音室、攝影棚、活動教室 |
| 課程涉及 | 場域、教室、設備、展覽、丈量、軟裝、軟體（硬體與軟體安裝相容性必查） |
| 開課時段 | 一三、二四、六、日（與課程類型同步評估） |
| 授課講師 | ①常態合作 ②儲備類型（案子多、非常態性）③客座講師 ④新老師（需風險評估） |
| 彈性空間 | 預留 **兩週** 彈性，預防天災或講師請假 |

其他排課考量：

- 參考「已註冊未上課」報表，掌握 **近六個月**，並觀察前一年。
- 注意報多科學員使用狀況（看「註冊學員課程預排需求報表」）。
- 區域排課類型評估分析、線上直播班納入考量。
- 教室使用分布要平衡（避免某層樓爆滿、某層樓閒置）。
- 全科班有專屬排課流程順序；特殊教室區域另行安排。

## 四、數據分析兩大盲點（已註冊未消耗）

依「已註冊未登記」人數決定開班前，先過濾：

1. **時間區間**：總人數（如 107 人）可能含 2022-2024 舊學員，回流意願低。**以近一年（2025-2026）數據為主**。
2. **上課狀態**：過濾同時報名多科、正忙於上其他課程的學生，才是真實可上課人數。

## 五、落點分析（中壢區案例）

檢視 5、6 月開課狀況發現的典型問題，作為各區自查範本：

- **時段不均**：開班集中假日（六、日），平日（一三、二四）幾乎沒課 → 平日晚上人流稀少、中心門可羅雀。
- **課程類型集中**：6 月多為繪畫類（漫畫、日本職人班），缺音樂、室內設計等其他職類平衡。
- **硬體考量**：中壢有 6F、7F、9F、12F 四個樓層，設備不同（Mac、PC、Wacom、攝影棚、錄音室），排課必須對齊硬體與軟體相容性。

## 六、行銷輔助標準化與 AI 應用

- **規格化內容**（10 餘項要素）：主副標、主視覺、適合對象、學習重點、課程大綱、老師介紹、QA 等。
- 學院需提供：**老師是誰 + 課綱** → 行銷部製作 **DM 稿（A4 版 + 手機版）+ 三支短影音**，分「課程招生版本」與「講座版本」。注意確認「多久前行銷部需提供給學院」的時程。
- **AI 賦能**：用 ChatGPT 等工具快速生成招生 DM（A4 + 手機版）。給對課綱與指令，AI 即可生成文案與視覺圖，減輕產品部負擔、加速行銷落實。

## 七、單元課（類講座）策略

目標：重新創造中心人流與氛圍。三種用途：服務、正式課暖身、特別招生。

- **3hr 免費課**：以話題性為主，獲取新名單（用哇哩勒卷兌換）。
- **6hr 收費課**：正式課程的「暖身」，解決學員等課空窗期。
- **登記流程改進**：移至學員專區由學生自行登記，不再透過業務手動配額，結合 LINE 對接服務。

## 八、排課管理平台

公司開發中，具備 **衝突偵測** 功能（教室 / 老師時段衝突，上線後預計衝突率歸零），業務端可即時查課。

## 九、招生衝班流程（客服主導版）

目標：提前發現招生風險、同步真實狀況、協助提升開班率。

| 階段 | 時點 | 負責 | 動作 |
|---|---|---|---|
| 1. 列出需衝班班級 | 開課前兩個月 | 客服 | 統計各班招生狀況、分析達成率、列出需提前衝班班級、呈報主管 |
| 2. 公告未登記名單 | 每週三 | 客服 | 更新未登記學員名單與班級招生狀況，公告於試算表/群組，通知業務回覆 |
| 3. 回覆學員狀態 | 每週五前 | 業務 | 回覆學員目前狀態（可上/不上/追蹤中/無回應/改班），更新最後聯絡紀錄 |
| 4. 回覆追蹤與能力驗收 | 每週五後 | 客服+主管 | 紀錄未回覆業務、提報主管與區主管、錄影驗收業務是否會銷售該課程、主管評分 |
| 5. 電話關懷未登記學員 | 開課前一個月 | 客服 | 針對「可再聯絡」名單電話關懷，確認意願與疑慮、協助解決問題、登記有意願學員 |
| 6. 業務支付協助獎金 | 促成報名後 | 業務+客服 | 客服電話促成學員報名上課，原業務支付客服 **300 元協助獎金** |
| 7. 安排衝班策略 | 每週固定追蹤 | 主管 | 檢視班級狀態與名單回覆，決定主推班級優先順序，調整資源與策略 |

### 試算表欄位

學員姓名、班級、業務、目前狀態、最後聯絡日、是否可再聯絡、備註。

### 學員 5 狀態

- **可上**：可安排報名上課
- **追蹤中**：業務持續洽談中
- **無回應**：尚未取得學員回覆
- **不上**：已確認無意願上課
- **改班**：學員改報其他班級

### 流程效益

提前兩個月預警掌握招生風險、固定回報節點資訊更即時、避免重複聯絡提升學員體驗、強化業務責任感與銷售能力、明確獎勵機制提高客服動機、主管提前決策資源有效分配。

## 十、後續行動方案

- 主管分析 2026 年 1-5 月開班狀況，找出修正點規劃下半年課程。
- 6 月第一週追蹤會議：主管提出各學院 **單元課規劃（每月每點約 3 門，共 9 門）** 及課程想法。

## 回答原則

- 回答排課問題時，先確認：開課時間點往前推 2 個月（衝班啟動）與 3-4 個月（預排判斷）。
- 開 B/C 班前，先檢查 A 班人數基數是否符合漏斗比例。
- 引用數據時提醒兩大盲點（時間區間、上課狀態）。
- 涉及月度業績目標、轉換率紅綠燈、a-e 類學員分類時，搭配 `hegrn-class-rush-plan`；涉及約訪話術時搭配 `hegrn-appointment-sop` / `hegrn-call-scripts`。
