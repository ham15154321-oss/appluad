## 二、最重要的 3 個帳號（保住它們 = 保住一切）

| 帳號 | 用途 | 萬一忘記密碼怎麼辦 |
|---|---|---|
| **Apple ID**（iCloud） | 保住網頁原始碼 | https://iforgot.apple.com 重設 |
| **Google 帳號**（Firebase） | 保住員工資料 | https://accounts.google.com/signin/recovery |
| **GitHub 帳號**（ham15154321-oss） | 保住部署、保住公開網站 | https://github.com/password_reset |

**強烈建議：**

1. 三個帳號**都啟用兩步驟驗證**（2FA），用 Google Authenticator 或 Authy app
2. **備份代碼**（recovery codes）印出來放抽屜或鎖到密碼管理器
3. 三個帳號**綁定到同一個手機號碼**，這樣換手機時通通可以救回來
4. 三個帳號的密碼存到 **1Password / Bitwarden** 之類的密碼管理器

## 五、平時的維護建議

### ✅ 每月一次

- 登入 Firebase Console → Firestore → 用「Export」匯出一份備份到 Google Cloud Storage（免費 5GB）
- 確認 GitHub Pages 部署沒爆掉：https://github.com/ham15154321-oss/appluad/actions 看綠勾

### ✅ 每季一次

### ✅ 員工流動時

- 不然換手機後可能被鎖

## 六、Firebase 專案的安全設定（強烈建議檢查）

打開 https://console.firebase.google.com/project/talent-map-e0c36/firestore/rules

確認你的 Firestore Rules 是這樣（**不是** `allow read, write: if true;` 開放讀寫，那會被全網路爬光）：

```
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /{document=**} {
      allow read, write: if request.auth != null;
    }
  }
}
```

意思：**必須要登入過匿名帳號才能讀寫**。本系統正是用這個方式（firebase-sync.js 第 1541 行 `auth.signInAnonymously()`）。

## 七、常見問題

### Q1：員工說登入後看不到自己的資料？

90% 機率是「Firebase Sync」還沒跑完。
- 請他重新整理頁面 1-2 次
- 還是不行 → 黃柏翰登入 → ⚙️ → 🔑 員工帳號管理 → 確認該員工的 characterName 對到正確的角色

### Q2：員工忘記密碼？

- 黃柏翰登入 → ⚙️ → 🔑 員工帳號管理 → 找到該員工 → 「🔧 重設密碼」→ 輸入新密碼 → 通知該員工
- 黃柏翰可以直接看到所有人明文密碼（這是你需求設計的）

### Q3：網站掛了？

1. 開 https://github.com/ham15154321-oss/appluad/actions 看最新一次部署
2. 紅色 ❌ → 點進去看錯誤訊息
3. 如果是 quota 爆掉 → 通常 24 小時自動恢復

### Q4：員工的 AI 對話為什麼會出現別人的名字？

不應該會。如果出現：
1. 員工 F12 → Console 看是否有 `[auth-guard] activeCharacterId 與登入主角不一致` 訊息（代表 guard 已修正）
2. 還是不對 → 員工登出再重新登入
3. 還是不對 → 通知 Claude / 找回 Cowork 修

### Q5：黃柏翰怎麼看所有員工的 AI 對話？

⚙️ 齒輪 → 🤖 AI 顧問中心 → 📊 總覽 — 看所有對話

可以過濾（主角、AI 工具、日期）、可以刪除（單筆 / 整段 / 過濾結果 / 🧹 清歷史歡迎詞）。

## 八、緊急情況下的「最後手段」

如果上述全部都掛了（極不可能），最後手段是：

1. **本地 IndexedDB 還可能有舊資料**
   - 在任何員工的 Chrome 上：F12 → Application → IndexedDB → 看 SpaceBaseDB、MotivArchiveDB、TrainingSalesDB、AiAdvisorDB → 右鍵 Export
   - 把這些 JSON 寄給你 → 你可以拿來重灌資料

2. **任何員工瀏覽器的 localStorage 也有資料**
   - 同上 F12 → Application → Local Storage → https://ham15154321-oss.github.io
   - 全選複製出來 → 寄給你

3. **聯絡 GitHub / Firebase 支援**
   - 帳號被盜可以追回
   - 資料誤刪有 30 天緩衝期
